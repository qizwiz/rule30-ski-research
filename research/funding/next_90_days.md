# Next 90 Days (Plain-Language Plan)

Goal: get funded by showing credible momentum, not by claiming the prize early.

## What we already have
- A real formal result (Inter-Basin theorem in Lean).
- Reproducible scripts and datasets.
- A coherent paper with clear proved vs observed claims.

## What we need next
- One new result that is clearly stronger than current baseline.
- A clean story outsiders can evaluate quickly.
- A repeatable update cadence (every 2 weeks).

## 30-day plan
1. Freeze exact statement for the Prize #3 track in plain language and formal language.
2. Freeze computational model assumptions (what counts as work, input/output, what algorithms are allowed).
3. Land 2-3 foundational lemmas in `prize3/lean` and keep them compiling.
4. Publish a short progress note with links to exact artifacts.

## 60-day plan
1. Produce first serious lower-bound argument draft under the frozen model.
2. Run internal adversarial review: list weakest assumptions and failure points.
3. Update paper appendix with progress and limitations.

## 90-day plan
1. Decide: either
- a) claim-ready proof path, or
- b) strong partial result + clear open gap.
2. Submit funding applications with a concrete milestone chart and burn-down list.

## Communication rules (to stay fundable)
- Never imply prize solved unless theorem is complete.
- Always separate:
  - proved
  - measured
  - conjectured
- Every claim must point to a file and command.

## Metrics to track biweekly
- Number of proved lemmas in `prize3/lean`
- Number of assumptions still unresolved
- Repro checks passing/not passing
- External feedback conversations completed
